# 1. BERT 要解决的问题

在 BERT 之前，很多语言模型采用单向建模：要么从左到右，要么从右到左。这样得到的表示只能利用一侧上下文，不利于做阅读理解、句间关系判断、文本分类这类“理解型”任务。BERT 的目标，就是学习一种**深度双向**的语言表示，使每个位置都能同时利用左、右上下文。

# 2. BERT 的核心思想

BERT 的全称是 **Bidirectional Encoder Representations from Transformers**。它本质上不是一个专门为单一任务设计的模型，而是一个 **先预训练、再微调** 的通用语言表示模型：先在大规模无标注文本上学习语言知识，再把同一个模型迁移到不同下游任务。BERT 采用的是 **Transformer Encoder**，重点不在生成，而在理解。

# 3. BERT 的方法框架

BERT 的方法可以概括为两步：

- 第一步：在无标注语料上做自监督预训练；
- 第二步：在具体任务上做监督微调。

它的关键不是“重新发明 Transformer”，而是把 **Transformer Encoder + 合适的预训练目标** 结合起来，让同一个编码器学到可迁移的语义表示。

# 4. BERT 的输入表示

BERT 的输入不是单一词向量，而是三部分相加：

- **Token Embedding**：词或子词本身的表示；
- **Segment Embedding**：区分句子 $A$ 和句子 $B$；
- **Position Embedding**：提供位置信息。

对第 $i$ 个 token，其输入可写为：
$$
x_i = e_i^{\text{token}} + e_i^{\text{segment}} + e_i^{\text{position}}
$$
此外，BERT 使用两个特殊标记：

- $[CLS]$：放在序列最前，用于句级任务；
- $[SEP]$：用于分隔句子

# 5. 为什么 BERT 能做双向建模

如果直接做双向语言模型，目标词可能“偷看到自己”，训练目标会失效。BERT 的解决方法是：**不要预测下一个词，而是随机遮住一些词，再预测被遮住的词**。这样模型只能依赖上下文来恢复目标词，从而实现真正的双向建模。

# 6. 预训练任务一：Masked Language Model（MLM）

## 6.1 任务定义

MLM 的做法是：从输入中随机选择一部分 token 作为预测目标，然后让模型根据上下文预测这些位置原本的词。若被遮盖位置集合记为 $M$，则 MLM 的目标可以写为：
$$
\mathcal{L}_{\text{MLM}} = - \sum_{i \in M} \log p(x_i \mid x_{\setminus M})
$$
这里，$x_i$ 表示第 $i$ 个位置原始 token，$x_{\setminus M}$ 表示其余可见上下文。

## 6.2  关键细节

BERT 不是把所有被选中的 token 都替换成 $[MASK]$。论文中的处理方式是：

- 随机选择大约 $15%$ 的 token 作为预测目标；
- 对这些位置：
  - $80%$ 替换为 $[MASK]$；
  - $10%$ 替换为随机 token；
  - $10%$ 保持原词不变。

这样做是为了缓解预训练与微调之间的不一致：因为下游任务输入中通常不会真的出现 $[MASK]$。

## 6.3  作用

MLM 让模型学到的不是“顺着生成一句话”，而是“在完整上下文中理解一个词”。因此，BERT 更擅长表示学习和语义理解，而不是自然文本生成。

# 7.预训练任务二：Next Sentence Prediction（NSP）

## 7.1 任务定义

NSP 的目标是判断句子 $B$ 是否是句子 $A$ 的下一句。输入形式为：[CLS] A [SEP] B [SEP]

模型用 $[CLS]$ 对应的最终表示做二分类。若标签记为 $y \in {\text{IsNext}, \text{NotNext}}$，则目标可写为：
$$
\mathcal{L}_{\text{NSP}} = - \log p(y \mid [CLS], A, B)
$$

## 7.2 数据构造

论文中的构造方式是：

- $50%$ 的样本中，$B$ 是 $A$ 的真实下一句；
- $50%$ 的样本中，$B$ 是语料中的随机句子。

这个任务的目的是让模型学习句子间关系，而不只是句内词关系。

## 7.3 作用

NSP 主要服务于需要句间推理的任务，例如自然语言推理、问答、句对分类。它让 BERT 不仅会“补词”，还会判断两句话在语义上是否连贯。

# 8. BERT 的总训练目标

BERT 在预训练阶段同时优化两个目标：
$$
\mathcal{L} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}
$$
因此，BERT 学到的表示同时包含：**词级上下文信息**、**句级关系信息**

# 9. BERT 训练完之后怎么用

BERT 的优势在于：**下游适配方式统一**。预训练完成后，只需在顶层接一个很小的任务头，就可以迁移到不同任务。

## 9.1 句子分类

取 $[CLS]$的最终表示 $C$，接一个分类层：
$$
p = \text{softmax}(W C + b)
$$
适用于情感分类、主题分类、自然语言推理等任务。

## 9.2 序列标注

对每个 token 的隐藏状态分别做分类，适用于命名实体识别、词性标注等任务。

## 9.3 抽取式问答

对每个位置预测它是否为答案起点或终点，因此依赖高质量的 token 级表示。

# 10. 最容易搞混的点

## 10.1 BERT 不是一个分类器

BERT 首先是一个**预训练语言表示模型**。分类、问答、序列标注只是它的下游使用方式。

## 11.2 BERT 的创新不是 Transformer 

Transformer 架构早于 BERT。BERT 的关键贡献是：用 **MLM** 实现深度双向预训练，再配合 **NSP** 和统一微调范式，把 Encoder 变成通用的语言表示学习器。

## 13.3 BERT 的强项是理解，不是自由生成

因为它的训练目标不是标准自回归语言模型，所以它不擅长像 GPT 那样自然地连续生成长文本。
